API – alle Daten als JSON

Sämtliche Termindaten sind offene Daten und lassen sich frei als JSON abrufen – ohne Anmeldung, ohne API-Schlüssel. Ideal, um eigene Anzeigen, Skripte oder Apps zu bauen.

Basis-URL{{ SITE_BASE_URL }}/api/v1/
FormatJSON (UTF-8)
Authentifizierungkeine
CORSAccess-Control-Allow-Origin: * – auch direkt aus dem Browser nutzbar
Rate-Limitfaire Nutzung; bei Überschreitung 429
Spezifikationopenapi.json (OpenAPI 3, maschinenlesbar)

Schnellstart: Termine einer Adresse holen

In drei Schritten von der Straße zu den Terminen als JSON:

1. Straße suchen → liefert eine id:

curl "{{ SITE_BASE_URL }}/api/v1/streets?q=beethoven"
{
  "results": [
    { "id": 123, "name": "Beethovenstraße", "district": null, "city": "Lübeck" }
  ]
}

2. Adresse auflösen (mit id und Hausnummer) → liefert eine dauerhafte public_id und die Bezirke je Abfallart:

curl "{{ SITE_BASE_URL }}/api/v1/address/resolve?street_id=123&house_number=11"
{
  "public_id": "azx87qx5bxra",
  "address": "Beethovenstraße 11, Lübeck",
  "zones": [
    { "code": "A", "waste_type": "gelber-sack" },
    { "code": "A", "waste_type": "restabfall" },
    { "code": "A", "waste_type": "bioabfall" },
    { "code": "J", "waste_type": "papier" }
  ]
}

3. Termine abrufen (mit der public_id):

curl "{{ SITE_BASE_URL }}/api/v1/addresses/{{ example_public_id }}/collections"
{
  "count": 105,
  "limit": 50,
  "offset": 0,
  "public_id": "{{ example_public_id }}",
  "results": [
    { "date": "2026-01-05", "waste_type": "gelber-sack", "zone": "A", "kind": "regular", "note": "" },
    { "date": "2026-01-08", "waste_type": "restabfall",  "zone": "A", "kind": "regular", "note": "" }
  ]
}

Die public_id ist stabil – einmal ermittelt, kannst du Schritt 3 dauerhaft wiederverwenden (auch über den Jahreswechsel hinweg).

Endpunkte

Methode & PfadBeschreibungParameter
GET /waste-types Alle Abfallarten (Slug, Name, Farbe, Icon). –
GET /streets Straßensuche. q (min. 2 Zeichen, erforderlich)
GET /address/resolve Adresse zu public_id + Bezirken auflösen. street_id oder street; house_number; optional district
GET /addresses/{public_id}/collections Abfuhrtermine einer Adresse. optional year, waste_type (Slugs, komma-getrennt), limit, offset
GET /zones/{code}/collections Termine eines Bezirks (Tourbuchstabe, z. B. A). optional year, waste_type, limit, offset
POST /reports Fehler melden. JSON: category, description, optional public_id
POST /corrections Korrekturvorschlag einreichen. JSON: kind, reason, …
GET /openapi.json Vollständige OpenAPI-3-Spezifikation. –

Direkt filtern

Nur den Gelben Sack für 2026:

curl "{{ SITE_BASE_URL }}/api/v1/addresses/{{ example_public_id }}/collections?waste_type=gelber-sack&year=2026"

Beispiele in JavaScript und Python

JavaScript (im Browser, dank CORS):

const r = await fetch("{{ SITE_BASE_URL }}/api/v1/addresses/{{ example_public_id }}/collections");
const data = await r.json();
console.log(data.results);

Python:

import requests
r = requests.get("{{ SITE_BASE_URL }}/api/v1/addresses/{{ example_public_id }}/collections")
for t in r.json()["results"]:
    print(t["date"], t["waste_type"])

Lieber ein Kalender-Abo?

Wer die Termine nur in seinen Kalender möchte, braucht keine API: Jede Adresse hat einen iCal-Feed (.ics), den man abonnieren kann – er aktualisiert sich automatisch. Suche einfach deine Adresse und wähle „Kalender abonnieren“.

Fehler & Antwortformat

Fehler kommen als JSON mit HTTP-Statuscode:

{ "error": { "code": "not_found", "message": "Adresse nicht gefunden." } }

Codes u. a.: bad_request (400), not_found (404), ambiguous (mehrdeutige Straße), rate_limited (429).

Nutzung: Offenes Bürgerprojekt, alle Angaben ohne Gewähr – verbindlich sind die Veröffentlichungen der Entsorgungsbetriebe Lübeck. Bei Weiterverwendung freuen wir uns über einen Hinweis auf abholtag.de. Bitte fair nutzen (Ergebnisse cachen statt in Schleife pollen).
